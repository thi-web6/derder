Pengaturan SSH pada Fedora dengan Autentikasi Berbasis Kunci

Dokumentasi ini memberikan panduan langkah demi langkah untuk mengatur Secure Shell (SSH) pada sistem operasi Fedora, dengan fokus pada autentikasi berbasis kunci. Tujuannya adalah memungkinkan akses aman tanpa kata sandi untuk pengguna tertentu, sambil mewajibkan kunci publik untuk pengguna lain, sehingga meningkatkan keamanan sistem.

Instalasi dan Aktivasi Server SSH

Sebelum mengatur autentikasi SSH, pastikan layanan OpenSSH Server terpasang dan aktif pada sistem Fedora. OpenSSH menyediakan protokol aman untuk komunikasi jarak jauh.

Langkah-langkah:

  1. Periksa keberadaan paket OpenSSH Server:

    Jalankan perintah berikut untuk memastikan openssh-server telah terpasang. Jika belum, perintah ini akan menginstalnya.

    sudo dnf install openssh-server

    Catatan: Jika muncul pesan bahwa paket sudah terpasang, lanjutkan ke langkah berikutnya.

  2. Aktifkan dan jalankan layanan SSH:

    Perintah berikut memastikan layanan sshd aktif saat booting dan berjalan saat ini.

    sudo systemctl enable sshd
sudo systemctl start sshd
  3. Verifikasi status layanan:

    Periksa apakah layanan sshd berjalan dengan benar.

    sudo systemctl status sshd

    Hasil yang diharapkan: Status active (running) menunjukkan layanan berjalan. Jika tidak, periksa log dengan sudo journalctl -u sshd.

Potensi masalah: Jika instalasi gagal, pastikan koneksi internet aktif dan repositori Fedora diatur dengan benar.

Pembuatan Pasangan Kunci SSH

Kunci SSH terdiri dari kunci publik dan privat, digunakan untuk autentikasi aman tanpa bergantung pada kata sandi. Proses ini dilakukan pada komputer klien (pengguna).

Langkah-langkah:

  1. Buat kunci SSH:

    Jalankan perintah berikut untuk menghasilkan kunci RSA dengan panjang 4096-bit, yang menawarkan keamanan tinggi.

    ssh-keygen -t rsa -b 4096
  2. Tentukan lokasi penyimpanan kunci:

    Saat diminta lokasi berkas, tekan Enter untuk menggunakan lokasi default (~/.ssh/id_rsa untuk kunci privat dan ~/.ssh/id_rsa.pub untuk kunci publik). Jika berkas sudah ada, konfirmasi untuk menimpa (y).

  3. Atur passphrase (opsional):

    Saat diminta passphrase, Anda memiliki dua pilihan:

    • Kosongkan untuk login tanpa kata sandi: Tekan Enter dua kali (untuk passphrase dan konfirmasi). Ini memungkinkan login otomatis, tetapi kurang aman jika kunci privat dicuri.
    • Isi untuk keamanan tambahan: Masukkan kata sandi yang kuat. Passphrase ini akan diminta setiap kali kunci digunakan, menambah lapisan perlindungan.

  4. Verifikasi hasil:

    Periksa berkas yang dihasilkan di direktori ~/.ssh dengan ls ~/.ssh. Anda akan melihat id_rsa dan id_rsa.pub.

Catatan penting: Simpan kunci privat (id_rsa) dengan aman dan jangan bagikan. Kunci publik (id_rsa.pub) dapat dibagikan ke server.

Penyalinan Kunci Publik ke Server

Kunci publik harus disalin ke server untuk mengaktifkan autentikasi berbasis kunci. Proses ini biasanya dilakukan menggunakan ssh-copy-id.

Langkah-langkah:

  1. Salin kunci publik:

    Jalankan perintah berikut dari komputer klien untuk menyalin kunci publik ke server.

    ssh-copy-id your_username@server_ip

    Ganti your_username dengan nama pengguna di server (misal, indodax) dan server_ip dengan alamat IP atau hostname server (misal, localhost atau 192.168.1.100).

  2. Masukkan kata sandi server:

    Saat diminta, masukkan kata sandi pengguna server. Ini diperlukan sekali sebelum autentikasi kunci aktif.

  3. Verifikasi akses:

    Coba login ke server untuk memastikan kunci berfungsi:

    ssh your_username@server_ip

    Jika berhasil, Anda akan masuk tanpa diminta kata sandi (jika passphrase dikosongkan) atau diminta passphrase (jika diatur).

  4. Untuk pengguna lain:

    Pengguna lain harus mengulangi langkah ini dari komputer mereka atau Anda dapat menambahkan kunci publik mereka secara manual ke ~/.ssh/authorized_keys di server:

    sudo nano ~other_user/.ssh/authorized_keys

    Tambahkan kunci publik, simpan, lalu atur izin:

    sudo chown other_user:other_user ~other_user/.ssh -R
sudo chmod 700 ~other_user/.ssh
sudo chmod 600 ~other_user/.ssh/authorized_keys

Potensi masalah: Jika ssh-copy-id gagal, pastikan server mengizinkan login kata sandi sementara (PasswordAuthentication yes di sshd_config) atau salin kunci secara manual.

Konfigurasi Server SSH untuk Autentikasi Kunci

Untuk meningkatkan keamanan, nonaktifkan login dengan kata sandi dan wajibkan autentikasi berbasis kunci dengan mengedit berkas /etc/ssh/sshd_config.

Langkah-langkah:

  1. Buat cadangan berkas konfigurasi:

    Sebelum mengedit, cadangkan berkas untuk mencegah kesalahan.

    sudo cp /etc/ssh/sshd_config /etc/ssh/sshd_config.bak
  2. Edit berkas konfigurasi:

    Buka berkas menggunakan editor teks seperti nano.

    sudo nano /etc/ssh/sshd_config
  3. Tambahkan atau ubah pengaturan:

    Tambahkan baris berikut di akhir berkas untuk mengaktifkan autentikasi kunci dan menonaktifkan login kata sandi.

    PubkeyAuthentication yes
PasswordAuthentication no
ChallengeResponseAuthentication no
AllowUsers your_username

    Penjelasan:

    • PubkeyAuthentication yes: Mengizinkan autentikasi berbasis kunci.
    • PasswordAuthentication no: Menonaktifkan login dengan kata sandi.
    • ChallengeResponseAuthentication no: Menonaktifkan metode autentikasi lain yang mungkin mem-bypass pengaturan.
    • AllowUsers your_username: (Opsional) Batasi akses hanya untuk pengguna tertentu (ganti your_username).

  4. Simpan dan mulai ulang layanan:

    Simpan berkas (Ctrl+O, Enter, Ctrl+X di nano), lalu mulai ulang sshd.

    sudo systemctl restart sshd
  5. Verifikasi konfigurasi:

    Coba login dari klien untuk memastikan hanya autentikasi kunci yang berfungsi:

    ssh your_username@server_ip

    Untuk menguji login kata sandi (harus gagal):

    ssh -o PubkeyAuthentication=no your_username@server_ip

Catatan penting: Jangan tutup sesi SSH saat ini sebelum menguji login di sesi baru untuk menghindari kehilangan akses jika konfigurasi salah.

Pengaturan Firewall dan SELinux

Fedora menggunakan firewalld untuk firewall dan SELinux untuk keamanan tambahan. Pastikan keduanya dikonfigurasi agar SSH berfungsi.

Firewall

  1. Buka port SSH:

    Pastikan port 22 (default SSH) diizinkan oleh firewalld.

    sudo firewall-cmd --permanent --add-service=ssh
sudo firewall-cmd --reload

    Catatan: Perintah --permanent memastikan aturan bertahan setelah reboot.

  2. Verifikasi aturan firewall:

    Periksa apakah layanan SSH aktif:

    sudo firewall-cmd --list-services

SELinux

  1. Uji mode permisif:

    Jika login SSH gagal, SELinux mungkin memblokir. Uji dengan mode permisif sementara:

    sudo setenforce 0

    Coba login lagi. Jika berhasil, SELinux perlu dikonfigurasi.

  2. Atur mode permisif permanen (opsional):

    Edit berkas SELinux untuk mengubah mode secara permanen:

    sudo sed -i 's/SELINUX=enforcing/SELINUX=permissive/' /etc/selinux/config

    Alternatif: Untuk tetap menggunakan mode enforcing, konfigurasikan aturan SELinux khusus dengan audit2allow berdasarkan log SELinux.

Potensi masalah: Jika firewall memblokir, periksa zona aktif dengan sudo firewall-cmd --get-active-zones dan pastikan layanan SSH diizinkan di zona tersebut.

Pemecahan Masalah

Jika terjadi masalah selama pengaturan atau login SSH, ikuti langkah berikut untuk mendiagnosis dan menyelesaikan.

Langkah-langkah:

  1. Periksa log SSH:

    Lihat log layanan SSH untuk mengetahui penyebab error.

    sudo journalctl -u sshd

    Contoh error: Authentication refused mungkin menunjukkan kunci salah atau PasswordAuthentication dinonaktifkan.

  2. Periksa izin berkas SSH:

    Pastikan izin direktori dan berkas SSH benar.

    chmod 700 ~/.ssh
chmod 600 ~/.ssh/authorized_keys
  3. Verifikasi kunci publik:

    Bandingkan kunci di ~/.ssh/authorized_keys server dengan ~/.ssh/id_rsa.pub klien untuk memastikan kecocokan.

  4. Uji koneksi:

    Gunakan mode verbose untuk detail error:

    ssh -v your_username@server_ip

Catatan penting: Jika semua gagal, kembalikan sshd_config dari cadangan (/etc/ssh/sshd_config.bak) dan uji ulang.
